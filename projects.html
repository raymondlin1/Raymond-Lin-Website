NachenBlaster

February, 2018
×
Primary Language: C++
Tools: Microsoft Visual Studio

NachenBlaster is a 2D Space-Invader-Style video game that I implemented for a school project. We were given a game engine, graphics and some starter header files that included functions that we had to implement. The game engine called a function, move(), many times within a certain time period in order to update the game.

A header file contained all of the possible game objects that derived from a single actor class. Each game object has different properties, such as movement speed, movement direction and patterns, health etc...

Each game object was derived from a single class. Then, each sub class inherited properties of the base class, but also added functionality specific to the particular class. For example, enemies and collectables were collidable, player projectiles could damage enemies but not the player himself or collectables, enemy projectiles could damage the player but not collectables and other enemies etc...

A game world class was used to keep track of all objects in the game, including the player, enemy aliens, moving stars in the background, and collectables. The game world class included a vector of game objects that could be added or destroyed based on the behavior of the game.

Cracked

February, 2018
×
Primary Language: C++
Tools: Microsoft Visual Studio

Cracked is a decoder for trivially encrypted message. We were given a message that was trivially encoded. Each character in the message string was XOR'd with the integer 42, to produce another string of characters. My task was to implement a program that finds all the possible decodings of the message, given a dictionary of valid English words.

The method of depth first search was used to find decodings. First, a tokenizer class was used in order to separate each encrypted word, and each word was placed in a vector. Then, for each encrypted word, we found English dictionary words that followed the same pattern and updated the current mapping in another translator class. Each new mapping was pushed onto the stack. This class included a vector stack of mappings from character to character. If there are conflicting mappings between characters, then we know the current mapping to be incorrect. We popped the mapping and used depth first search in order to look for a new correct mapping based on another English word that fit the pattern.

Once we find a valid mapping, another class was called in order to decrypt the message and print it out to the screen. We then continued to find more valid mappings.
